# .claude/tasks/{team-or-session-id}/{n}.json

チームまたはセッションに紐づくタスク定義。番号付きJSONファイルで管理される。

---

```json
{
  "id": "<task-number>",
  "subject": "<task-subject>",
  "description": "<task-description>",
  "status": "<in_progress | completed>",
  "blocks": [],
  "blockedBy": [],
  "metadata": {
    "_internal": true
  }
}
```

## フィールド説明

| フィールド    | 型     | 説明                                                     |
| ------------- | ------ | -------------------------------------------------------- |
| `id`          | string | タスク番号（ファイル名の `{n}` と一致）                  |
| `subject`     | string | タスクの件名・対象（エージェント名やコンポーネント名等） |
| `description` | string | タスクの詳細説明                                         |
| `status`      | string | ステータス（`in_progress`, `completed`）                 |
| `blocks`      | array  | このタスクがブロックしている他タスクのID                 |
| `blockedBy`   | array  | このタスクをブロックしている他タスクのID                 |
| `metadata`    | object | メタデータ。`_internal: true` は内部タスク               |

## 補足ファイル

同ディレクトリに以下も存在する:

- `.lock` — ロックファイル（排他制御用）
- `.highwatermark` — 最新タスク番号を記録
