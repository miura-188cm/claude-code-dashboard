# .claude/debug/{session-id}.txt

セッションごとの詳細デバッグログ。テキスト形式、1行1エントリ。

---

## フォーマット

```
<ISO8601-timestamp> [<level>] <message>
```

```
2026-03-06T03:46:07.028Z [DEBUG] [onSubmit] early return: suggestions showing (count=2)
2026-03-06T03:46:07.031Z [DEBUG] detectFileEncoding failed for expected reason: ENOENT
2026-03-06T03:46:07.034Z [DEBUG] Getting matching hook commands for SessionStart with query: clear
2026-03-06T03:46:07.034Z [DEBUG] Found 0 hook matchers in settings
2026-03-06T03:46:07.070Z [DEBUG] AutoUpdaterWrapper: Installation type: npm-global
```

## 主なログパターン

| パターン | 説明 | 分析用途 |
|---|---|---|
| `[onSubmit]` | ユーザー入力イベント | 入力タイミング分析 |
| `Getting matching hook commands for <event>` | Hook検索（`PreToolUse`, `SessionStart`, `SessionEnd`） | Hook利用状況 |
| `Matched N unique hooks` | Hookマッチ結果 | Hookヒット率 |
| `LSP Diagnostics` | LSP診断情報のリセット・追跡 | エディタ連携状況 |
| `AutoUpdaterWrapper: Installation type` | インストール種別（`npm-global` 等） | 環境情報 |
| `claudeai-mcp` | MCPサーバー接続・ゲート判定 | MCP利用状況 |
| `FileHistory: Added snapshot` | ファイルスナップショット記録 | ファイル変更追跡 |
| `Full reset` | 画面リセット | UI状態変化 |
| `High write ratio` | 画面書き込み比率 | パフォーマンス |

## 備考

- ログレベルは主に `[DEBUG]`
- 構造化されていないテキストなので、正規表現でのパース が必要
