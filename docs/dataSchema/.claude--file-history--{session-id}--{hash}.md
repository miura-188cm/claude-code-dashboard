# .claude/file-history/{session-id}/{hash}@{version}

ファイル変更のスナップショット。セッション中に変更されたファイルのバックアップで、undo/restore 用。

---

## ディレクトリ構造

```
.claude/file-history/
└── <session-uuid>/
    ├── <content-hash>@v1
    ├── <content-hash>@v2
    └── ...
```

## ファイル名パターン

```
<16桁の16進ハッシュ>@v<version-number>
```

例: `16f561ef2dd36bce@v2`, `2326d4deed66062f@v1`

## 備考

- バイナリ形式のファイルコンテンツバックアップ
- セッションJSONL内の `file-history-snapshot` タイプと連携する
- `trackedFileBackups` の値がこれらのファイルハッシュを参照する
- ダッシュボード分析では「変更されたファイル数」のカウント程度に使用可能
